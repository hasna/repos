# open-repos "goat" — durable agent brief (account009)

Investigate how we structure repos across ALL Hasna GitHub orgs (hasna, hasnaxyz,
hasnatools, hasnastudio) and extend open-repos accordingly.

## Investigate
- Cross-org repo taxonomy: prefixes (iapp-, service-, skill-, hook-, connect-,
  platform-, project-, scaffold-, open-), folder conventions, topics, naming.
- How repos map to local Workspace paths + which machine they're developed on.

## Build into open-repos
1. Per-repo: select the PRIMARY machine it's developed on (fleet: machine001-011,
   apple0x, spark01/02) + the local path on that machine.
2. Integration with open-projects: link which projects belong to a repo (repo↔project).
3. Semantic search across repo metadata (name, description, topics, README, lang).
4. Keep CLI + HTTP + MCP surfaces in sync per the four-primitive convention.

Ship via the package publish flow; update local install after publish.
